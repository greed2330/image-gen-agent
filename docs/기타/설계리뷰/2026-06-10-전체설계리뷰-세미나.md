# 전체 설계 리뷰 & 자체 세미나 (2026-06-10)

> 작성: Opus. 범위: 설계문서 00~08 + 체크리스트 + 세션로그 001/002 + 코드 스켈레톤(Phase 2).
> 목적: Phase 1 진입 전, 전제 설계의 정합성·리스크·미정의 결정을 비판적으로 점검.
> 성격: 단정이 아니라 **리뷰**. 발견 = (A) 정합성 버그(즉시 수정), (B) 구조적 리스크(토론), (C) 미정의 결정(오너 판단).

## 1. 한눈에 보는 아키텍처 (재구성)

```
한국어 발화/레퍼런스
  → ① Intent (qwen3:14b SFW / abliterated:14b NSFW)   [VRAM ~9GB]
  → ② Router (workflow·checkpoint·lora 선택)
  → ③ Compile  [이해]LLM 씨앗 → [합성]TIPO → [검증]allowlist   [LLM+TIPO ~10GB]
  → ④ Param (프리셋 저점 + LLM nudge + hard clamp)
  → (전환: LLM 전부 언로드)
  → ⑤ ComfyUI 생성 (SDXL 8GB 단독 / Chroma 13GB 단독)
  → (전환: /free → 비전 로드)
  → ⑥ Critic (qwen3-vl-abliterated:8b)  재시도 시 ⑤로 스왑 왕복   [6.1GB]
  → 결과 → mem0 기록 → ①로 순환
```

**정수(불변 핵심)**: ① 레이어 분리(이해/합성/검증) + ② 프리셋 저점 + 하드클램프 + ③ allowlist 결정론적 안전망. 이 셋이 "저능 LLM으로도 쓰레기 안 나오게" 만드는 골격.

---

## 2. 강점 (잘 설계된 점)

1. **이해≠합성 분리 + TIPO 위임** — 범용 LLM 어휘 부담↓ → hallucination 구조적 감소. danbooru 공용어라 포맷 정의 불필요. (07 §1-A)
2. **저점=프리셋 / 고점=LLM nudge + hard clamp** — 저능 LLM이 파라미터를 망쳐도 클램프가 쓰레기값 차단. 능력에 비례해 품질이 *우아하게* 저하. (07 §3)
3. **allowlist 결정론적 안전망** — 확률적 합성 위에 "하드 0% 보장" 백스톱. 옳은 보강.
4. **VRAM 단일점유 + 언로드 순서(ollama 버그 인지)** — 현실 기반. 탁상공론 아님. (02)
5. **의사결정 규율** — 모든 결정에 "왜 + 버린 대안 + 검증/미검증 마킹". 재논의 비용 제거. (드묾, 유지할 것)

## 4. 구조적 리스크 (B — 토론 필요)

### B1. 다단계 스왑 = 정의되지 않은 지연 예산 ★
- 1회 생성에 모델이 여러 번 교체됨(두뇌→이미지→비전, 재시도 시 반복).
- RAM 캐싱(64GB)이 디스크 재로드는 없애도 **VRAM↔RAM 스왑 자체 시간**(9GB+8GB 왕복)은 남음.
- **어디에도 "생성 1회 ≤ X초" 목표가 없음.** 스왑 중심 구조인데 지연 예산이 없으면 설계를 검증할 기준이 없음.

### B2. 런타임 거부 탐지 메커니즘 부재 ★
- 07 §1-A 원칙 2: "qwen3:14b가 거부하면 abliterated로 폴백".
- 그런데 **"거부했다"를 코드가 어떻게 판정하나?** 거부 문구 패턴? 빈 JSON? 저품질 파싱과 거부를 어떻게 구분?
- 폴백 트리거가 정의 안 됨 → 분리 원칙이 런타임에 작동 안 할 수 있음.

### B3. TIPO 단일 의존 — Plan B 부재 ★
- hallucination 대책 전체가 TIPO에 걸려 있는데 **NSFW 품질 미검증**(07/03/08 모두 인정).
- H1(TIPO가 NSFW 잘 확장)이 깨지면 → 합성이 저능 abliterated의 raw 태그 생성으로 회귀 = 피하려던 바로 그 상태.
- 08은 "매핑 확대"를 대안으로 들지만 그것도 결국 LLM 의존. **진짜 Plan B(예: DanTagGen 교체, 수동 사전 부분 복원)가 문서화 안 됨.**

### B4. 하이브리드의 사각지대 — 복잡한 NSFW 의도
- 01 라우팅: 복잡 추론→클라우드, NSFW→로컬무검열.
- **복잡한 NSFW 의도**는? 클라우드 거부 → 저능 abliterated만 남음. 하이브리드 이점이 가장 필요한 지점에서 증발.
- 명시적으로 인정하고 "이 경우 품질 한계 수용" 또는 완화책 필요.

### B5. mem0 추출 모델 미배정
- 04: 메모리 자동 추출 LLM이 NSFW 컨텍스트 다루려면 무검열이어야 함(인정).
- 그런데 **03 모델 배정표에 "mem0 추출용" 역할이 없음.** 어느 모델? 추출 시점 VRAM 점유는? 미정의.

### B6. Critic 루프 — 비용 대비 가치 불확실
- 비전 모델은 손가락 개수 세기 자체가 불안정. 게다가 시드/파라미터 안 바꾸고 재생성하면 같은 오류 재현 가능.
- 02 상태머신은 Critic을 코어로 그리지만 체크리스트는 Phase 3. **코어 vs 지연 위상 불일치.** → Phase 3로 명확히 격하 권장.
