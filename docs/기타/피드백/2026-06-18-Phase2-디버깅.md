# Phase 2 구현 디버깅 기록 (2026-06-18)

## 버그 1: tipo_client — parse_tipo_request 시그니처 불일치

**증상**: `parse_tipo_request() got an unexpected keyword argument 'specials'`

**원인**: session_005 메모(prompt.md)에 기록된 TIPO 호출 방식이 실제 설치된 버전과 달랐음. 실제 API는 `tag_map` dict 하나를 첫 인자로 받는 방식.

**실제 시그니처**: `parse_tipo_request(tag_map, nl_prompt='', expand_tags=True, ...)`
- `tag_map` 키: `rating`, `special`, `general`, `quality`, `characters`, `copyrights`, `artist`, `meta`
- 반환: `(meta, operations, general_str, nl_prompt)`
- `tipo_runner` 반환: `(parsed_dict, timing)` — `parsed.get("special") + parsed.get("general")`이 확장 태그

**교훈**: 라이브러리 API는 코드 내 `inspect.getsource`로 직접 확인. 문서나 메모에 의존하면 버전 차이로 깨진다.

## 버그 2: ollama_client — /api/chat의 system 파라미터

**증상**: 모델이 한국어를 "garbled"로 판단하거나 의도 추출 거부

**원인**: `/api/chat`에 `system` 필드를 top-level JSON으로 보냈는데, 시스템 프롬프트가 적용 안 됨. `/api/generate`는 `system` top-level을 지원하지만 `/api/chat`은 messages 배열 내 `role: "system"` 메시지로 넣어야 함.

**수정**: `OllamaClient.chat`에서 `system` 인자를 `[{"role":"system","content":system}] + messages`로 prepend.

**교훈**: ollama 두 엔드포인트(`/api/generate` vs `/api/chat`)의 system 파라미터 처리 방식이 다름. chat은 messages 형식이 표준.

## 버그 3: tipo_client — nsfw rating 고정

**증상**: SFW 요청("분홍 머리 소녀")에 explicit 태그(`pussy`, `anus`) 생성

**원인**: TIPO의 `tag_map["rating"]`을 항상 `["explicit"]`로 하드코딩. TIPO는 rating에 따라 공존 태그 분포가 다름.

**수정**: `nsfw_level` 파라미터를 `expand_tags`에 추가, 0→"safe", 1→"sensitive", 2→"explicit"으로 매핑.

## 버그 4: PowerShell HTTP body 인코딩 (환경 문제, 코드 버그 아님)

**증상**: 서버가 한국어를 `??`로 수신, 모델이 의미 추출 불가

**원인**: PowerShell `Invoke-RestMethod`에 `-Body $string`으로 전달 시 Windows 시스템 인코딩(CP949 계열)으로 변환됨. UTF-8 아님.

**테스트 시 올바른 방법**:
```powershell
$bytes = [System.Text.Encoding]::UTF8.GetBytes($jsonString)
Invoke-RestMethod -Uri "..." -Method POST -ContentType "application/json; charset=utf-8" -Body $bytes
```

**프런트엔드/클라이언트 코드에는 무관**: httpx, fetch, axios는 기본 UTF-8 인코딩.

## qwen3 thinking 모드 처리

**증상**: `format="json"` 옵션 → 한국어 거부 오류 JSON 반환. thinking 없이도 empty JSON 반환.

**해결**:
1. 시스템 프롬프트에 `/no_think` 접두사 추가 (qwen3 thinking 비활성화)
2. few-shot 예시를 "Input → Output" 형식으로 명확히
3. `_extract_json`을 중괄호 깊이 추적 방식으로 강화 (thinking 블록 내 JSON 혼선 방지)

**주의**: `format="json"` 옵션은 한국어 입력 시 qwen3에서 거부 응답을 유발함 — 사용 금지.
