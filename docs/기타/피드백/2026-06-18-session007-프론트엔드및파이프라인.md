## 실수 1: +버튼을 stub으로만 남긴 채 설명 없이 배포

**무엇**: 프론트엔드 composer의 +버튼에 `title="이미지 첨부 (준비 중)"` 외에 아무 동작도 없이 배포.
**왜 문제**: 오너가 눌러봤을 때 아무 반응 없음 → "왜 아무것도 안 만들어놨어?" 지적.
**근본 원인**: Phase 3 미완 기능(img2img)과 연결된 기능이라 후순위로 둔 것인데, 최소한 UX(파일 선택 + 프리뷰)는 구현했어야 함. 백엔드 미구현과 프론트엔드 UX는 분리할 수 있었다.
**재발 방지**: stub 버튼을 배포할 때는 (1) 눌렸을 때 "Phase 3 준비 중" 모달이라도 띄우거나 (2) 버튼 자체를 disabled 처리하고 tooltip 명시. 아무 반응 없는 클릭 가능 요소는 배포 금지.

## 실수 2: IntentParser에 대화 맥락을 주입하지 않음

**무엇**: 각 생성 요청이 독립적으로 파싱됨. "백호 수인 소녀" 확립 후 "스포츠 브라로 바꿔줘" 요청 시 백호 수인 특성 전부 소실.
**왜 문제**: 오너가 3번의 프롬프트를 반복해서 입력해야 했고, 여전히 원하는 결과가 안 나옴.
**근본 원인**: 채팅 인터페이스인데 대화 맥락을 LLM에 전달하는 설계를 빠뜨림. GenRequest에 history 필드 자체가 없었음.
**수정**: GenRequest에 `history: list[HistoryMessage]` 추가, 프론트엔드에서 이전 메시지 전달, `_build_messages()`에서 LLM messages에 포함.
**재발 방지**: 채팅 형태의 인터페이스를 구현할 때는 항상 "이 LLM 호출이 이전 대화를 알고 있는가?"를 체크리스트에 포함.

## 실수 3: 수인(kemonomimi) danbooru 태그 번역 미흡

**무엇**: "백호 수인" → `white tiger`(동물 태그)로 번역. 결과물이 수인 소녀가 아닌 동물/쿠션.
**왜 문제**: danbooru에서 `white tiger`는 동물 그 자체를 가리킴. 수인은 `tiger ears`, `tiger tail`, `kemonomimi`가 올바른 태그.
**근본 원인**: 시스템 프롬프트 예시에 수인/kemonomimi 케이스가 없었음.
**수정**: 시스템 프롬프트에 백호 수인·늑대 수인 예시 추가, 규칙에 "동물 이름 단독 사용 금지 → {species} ears/tail 형태 사용" 명시.
**재발 방지**: 특수 장르(수인, 마인, 사이보그 등) 태그는 danbooru 표기 규칙 확인 후 few-shot 예시에 반드시 포함.
