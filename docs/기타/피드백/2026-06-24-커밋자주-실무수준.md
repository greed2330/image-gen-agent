# 피드백 — 2026-06-24 — 커밋 정책 변경 (실무 수준 잦은 커밋)

## 무엇
오너 신규 상시 지침: **앞으로 git 커밋을 실무 수준으로 자주** 할 것. 매 커밋마다 요청을 기다리지 말고, 논리적 작업 단위마다 자율적으로 커밋한다.

## 왜
- 기존 CLAUDE.md 규칙은 "오너가 요청할 때만 커밋/푸시" → 한 세션의 작업이 통째로 미커밋 상태로 쌓였음(session_011도 끝에 가서야 커밋).
- 실무에서는 논리 단위마다 작은 커밋을 쌓는다: 이력 추적·되돌리기·리뷰가 쉬워짐. 오너가 이 실무 관행을 원함.

## 어떻게 적용 (재발 방지 = 행동 규칙)
- **커밋: 자율·빈번.** 논리적 작업 단위가 끝나면 즉시 커밋. 권장 ~3파일 또는 1개 논리 변경.
- **단위 분리.** 한 커밋엔 한 논리 변경만. 무관한 변경(예: 버그수정 + 새 기능 + 문서)을 한 커밋에 섞지 않는다.
- **푸시는 여전히 오너 요청 시에만** (외부 노출 행위). 커밋은 로컬에 쌓아둔다.
- 컨벤션 타입 접두사 유지: `feat/fix/refactor/docs/chore`, co-author 푸터 유지.
- CLAUDE.md "Git / Workflow" 섹션에 반영 완료.
