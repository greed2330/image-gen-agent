# 기타 문서 폴더 안내

작업 과정에서 나오는 기록을 종류별로 보관. **실수 재발 방지**가 핵심 목적.

## 하위 폴더

| 폴더 | 용도 | 작성 시점 |
|------|------|-----------|
| `피드백/` | 이전에 했던 실수·잘못된 접근을 기록해 **재발 방지**. 사용자 피드백/교정도 여기. 파일명: `YYYY-MM-DD-주제.md` | 실수나 교정이 발생할 때마다 |
| `디버깅기록/` | 버그 증상 → 원인 → 해결 과정 기록. 같은 버그 재발 시 빠른 참조 | 비자명한 버그를 해결할 때 |
| `코드리뷰/` | 코드 리뷰 지적사항·개선점 기록 | 리뷰 수행 시 |
| `외부레퍼런스/` | 참고한 외부 자료(블로그/레포/CLAUDE.md 레퍼런스 등) 스냅샷·링크 정리 | 자료 수집 시 |

## 작성 원칙

- 한 파일 = 한 사건/주제. 짧고 구체적으로.
- 피드백 문서는 반드시 **무엇이 문제였나 / 왜 / 앞으로 어떻게**를 포함.
- 작업 현황 자체는 여기 말고 `../작업기록/체크리스트.md`에 반영.
