# 13. 이미지 다운로드 버튼

> 작성: 2026-06-18 (Opus 설계) · 구현: Sonnet
> 이 문서는 **구현 지시서**다. Sonnet은 추론 없이 아래 명세 그대로 구현한다.
> 독립 기능 — 다른 문서와 의존 없음. 소규모.

## 목적

생성된 이미지를 **버튼 한 번으로 브라우저 다운로드 폴더에 저장.** 우클릭 → 다른 이름으로 저장 불필요.

## 설계 결정 — 백엔드 Content-Disposition (왜 / 버린 대안)

- **채택:** 이미지 라우트에 `?download=1` 쿼리 → `Content-Disposition: attachment` 헤더. 브라우저가 **교차 출처(8000↔3000)에서도 무조건 다운로드**. 프론트는 링크 클릭만.
- **버린 대안 — 프론트 `<a download>`:** `download` 속성은 **교차 출처 리소스에 무시됨**(브라우저 보안). 이미지가 localhost:8000, 앱이 localhost:3000이라 그냥 열려버림. → 안 됨.
- **버린 대안 — fetch blob + objectURL:** 동작하지만 이미지 전체를 메모리에 올림 + 코드 복잡. 헤더 방식이 더 단순. (대용량 다발 다운로드가 생기면 그때 재고.)

> `FileResponse(path, filename=...)`가 자동으로 `Content-Disposition: attachment; filename=...`를 설정한다.

# 구현

## A. `backend/app/api/routes_images.py` — download 쿼리 추가

```python
@router.get("/{filename}")
async def serve_image(filename: str, download: bool = False) -> FileResponse:
    """Serve a generated image. download=1 → 첨부(다운로드)로 응답."""
    if "/" in filename or "\\" in filename or ".." in filename:
        raise HTTPException(status_code=400, detail="invalid filename")
    path = Path(settings.comfyui_output_dir) / filename
    if not path.exists():
        raise HTTPException(status_code=404, detail="image not found")
    if download:
        return FileResponse(path, media_type="image/png", filename=filename)
    return FileResponse(path, media_type="image/png")
```

> 경로 탐색 방어(기존)는 그대로 유지. `download` 외 동작 변화 없음(회귀 없음).

## B. 프론트엔드

> ⚠️ `frontend/AGENTS.md`: 이 Next.js는 학습 데이터와 다름. 수정 전 `node_modules/next/dist/docs/` 확인.

`frontend/lib/api.ts`에 헬퍼 추가:

```typescript
export function downloadUrl(imagePath: string): string {
  return `${imageUrl(imagePath)}?download=1`;
}
```

UI: 생성된 이미지(GenCard) 및 라이트박스에 **다운로드 버튼** 추가. 클릭 시 `downloadUrl(path)`로 다운로드 트리거. 프로그래매틱 앵커 클릭:

```typescript
function downloadImage(imagePath: string) {
  const a = document.createElement("a");
  a.href = downloadUrl(imagePath);
  a.click();   // Content-Disposition: attachment 이므로 다운로드 폴더로 저장됨
}
```

- 위치: GenCard 호버 오버레이 또는 라이트박스 상단. 라이트박스(클릭 확대)에 두는 것을 우선.
- 클릭 가능 요소이므로 즉시 동작해야 함(stub 금지 — `[[no-dead-buttons]]` 원칙).

## 다운로드 파일명 (참고)

v1은 원본 파일명(`output_xxxxx_.png`) 그대로. 추후 doc 10 생성기록(시각·캐릭터 태그)으로 의미 있는 이름(`백호수인_20260618.png`) 부여 가능 — 지금은 과설계 방지로 보류.

## 검증

```
1. GET /images/{f}?download=1 → 응답 헤더에 Content-Disposition: attachment 확인
2. 라이트박스에서 다운로드 버튼 클릭 → 브라우저 다운로드 폴더에 .png 저장
3. 일반 표시(다운로드 안 함)는 기존대로 inline 렌더 (회귀 없음)
```
