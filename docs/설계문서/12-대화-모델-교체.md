> 독립 기능 — 09/10/11과 의존 없음. 아무 때나 구현 가능.

## 목적 / 범위

ollama에 설치된 모델을 **능동 탐색**해서, **대화(의도 해석) LLM**을 런타임에 교체한다.

**범위 한정:**
- 교체 대상 = **메인 대화 모델**(SAFE/SUGGESTIVE 의도 해석용, 현재 `settings.llm_main_model`).
- **교체 안 하는 것(고정):**
  - **NSFW explicit 폴백 모델**(`settings.llm_nsfw_model`, abliterated) — **NSFW 정책상 검열 모델로 바뀌면 안 됨**. 사용자 선택 대상에서 제외하고 abliterated 고정.
  - TIPO, 임베딩(`nomic-embed-text`), VL 모델 — 이번 범위 밖.

> **왜 메인만:** 오너 요청("대화 모델 정도는 교체"). NSFW 합성 모델을 사용자가 검열 모델로 바꾸면 무검열 프로젝트 원칙(CLAUDE.md)이 깨지므로 의도적으로 잠근다.

## 현재 상태

- `OllamaClient`에 모델 목록 조회 없음. ollama `GET /api/tags`가 설치 모델 반환(확인됨: qwen3:14b, qwen3:4b, abliterated:14b, qwen3-vl:8b, qwen2.5-coder:14b, gemma4:12b, nomic-embed-text).
- `IntentParser._ollama_model_for()`가 `settings.llm_main_model` 직접 참조.
- 런타임 가변 설정을 담을 곳 없음.

# 구현

## A. 런타임 설정 저장 — `backend/app/services/runtime_config.py` (신규)

도메인 데이터(doc 10 DB)와 분리된 **가변 앱 설정**. 작은 JSON 파일. 재시작에도 유지.

```python
import json
from pathlib import Path
from app.config import settings

_PATH = Path(__file__).parent.parent.parent / "data" / "runtime_settings.json"


class RuntimeConfig:
    def __init__(self) -> None:
        self._data: dict = {}

    def init(self) -> None:
        _PATH.parent.mkdir(parents=True, exist_ok=True)
        if _PATH.exists():
            self._data = json.loads(_PATH.read_text(encoding="utf-8"))

    def get_chat_model(self) -> str:
        return self._data.get("chat_model") or settings.llm_main_model

    def set_chat_model(self, model: str) -> None:
        self._data["chat_model"] = model
        _PATH.write_text(json.dumps(self._data, ensure_ascii=False, indent=2), encoding="utf-8")
```

> 기본값은 `settings.llm_main_model`. 사용자가 고르기 전까지 기존 동작 그대로(회귀 없음).

## B. `ollama_client.py` — 모델 탐색

```python
async def list_models(self) -> list[str]:
    """ollama에 설치된 모델 이름 목록 (GET /api/tags)."""
    async with httpx.AsyncClient() as client:
        resp = await client.get(f"{self._base}/api/tags", timeout=10)
        resp.raise_for_status()
        data = resp.json()
    return [m["name"] for m in data.get("models", [])]
```

## C. `intent_parser.py` — 런타임 모델 적용

`IntentParser`에 `runtime: RuntimeConfig` 주입. `_ollama_model_for` 변경:

```python
def _ollama_model_for(self, level: NsfwLevel) -> str:
    if level == NsfwLevel.EXPLICIT:
        return settings.llm_nsfw_model          # 고정 (NSFW 정책)
    return self._runtime.get_chat_model()        # 사용자 선택 가능
```

## D. 라우트 — `backend/app/api/routes_models.py` (신규)

```python
router = APIRouter(prefix="/models", tags=["models"])

@router.get("")
async def list_models() -> dict:
    """탐색된 모델 + 현재 대화 모델. 임베딩 모델은 채팅 후보에서 제외."""
    from app.deps import ollama, runtime_config
    available = [m for m in await ollama.list_models() if "embed" not in m.lower()]
    return {"available": available, "current": runtime_config.get_chat_model()}

@router.put("/chat")
async def set_chat_model(body: dict) -> dict:
    from app.deps import ollama, runtime_config
    model = body.get("model")
    installed = await ollama.list_models()
    if model not in installed:                    # 존재 검증
        raise HTTPException(status_code=400, detail=f"model not installed: {model}")
    runtime_config.set_chat_model(model)
    logger.info("chat model switched -> %s", model)
    return {"current": model}
```

`main.py`에 `models_router` 등록.

> 임베딩 모델(`*embed*`)만 후보에서 제외. coder/vl 등은 채팅 가능하므로 남겨 둠(사용자 자유).

## VRAM 참고

모델 교체 후 다음 의도 해석 때 새 모델이 ollama에 로드됨(ollama가 keep_alive로 관리). 생성 직전 `vram_manager`가 LLM 언로드를 수행하므로 충돌 없음. 교체 시 이전 모델 강제 언로드는 불필요(선택적 최적화).

## 검증

```
1. GET /models → {available:[...embed 제외...], current:"qwen3:14b"}
2. PUT /models/chat {"model":"gemma4:12b"} → {current:"gemma4:12b"}, runtime_settings.json 갱신
3. PUT /models/chat {"model":"없는모델"} → 400
4. 백엔드 재시작 → GET /models의 current가 gemma4:12b 유지
5. dry-run 생성 → 백엔드 로그에서 의도 해석이 선택 모델로 호출됨 확인
6. EXPLICIT 요청 → 여전히 abliterated 사용(고정) 확인
7. tests/test_clients.py: list_models mock 1개. routes 검증(set은 존재검증 실패 케이스 포함).
```
