## 전체 파이프라인

GPT 이미지 생성이 내부적으로 숨기고 있는 단계를 명시적으로 분해한 구조.

```
[사용자 발화(한국어) / 레퍼런스 이미지]
        │
        ▼
┌──────────────────────────────────────────────┐
│ ① 의도 해석 (Intent Parser)                    │
│    - 무엇을? 스타일/구도/용도?                  │
│    - 레퍼런스 이미지 → 포즈만? 스타일만? 인물만?│
│    - (이미지 입력 시) qwen3-vl-abliterated:8b 분석│
└──────────────────────────────────────────────┘
        │  구조화된 intent (JSON)
        ▼
┌──────────────────────────────────────────────┐
│ ② 작업 라우팅 (Workflow Router)                │
│    - txt2img / img2img / inpaint / controlnet  │
│      / ipadapter 중 무엇을 쓸지 결정            │
│    - 어떤 체크포인트 / LoRA 조합을 쓸지 결정     │
└──────────────────────────────────────────────┘
        │
        ▼
┌──────────────────────────────────────────────┐
│ ③ 프롬프트 컴파일 (Prompt Compiler)            │
│    - [이해] LLM: 한국어 → 거친 씨앗 태그        │
│    - [합성] TIPO: 씨앗 → 완성 danbooru 프롬프트 │
│      (확장·조합·정렬, danbooru 전용 모델)       │
│    - [검증] allowlist 대조 → 비존재 태그 제거    │
│    - 네거티브 / 퀄리티 토큰 / 가중치 부여        │
│    (Illustrious=태그 / Flux·Chroma=자연어 분기) │
└──────────────────────────────────────────────┘
        │
        ▼
┌──────────────────────────────────────────────┐
│ ④ 파라미터 결정 (Param Resolver)               │
│    - steps, cfg, sampler, scheduler            │
│    - 해상도(종횡비), denoise strength           │
│    - 모델별 프리셋 + intent 기반 조정            │
└──────────────────────────────────────────────┘
        │  ComfyUI workflow graph (JSON)
        ▼
┌──────────────────────────────────────────────┐
│ ⑤ 실행 엔진 (ComfyUI, HTTP API)                │
│    - 워크플로우 JSON 제출 → 이미지 생성          │
└──────────────────────────────────────────────┘
        │
        ▼
┌──────────────────────────────────────────────┐
│ ⑥ 자기 평가 루프 (Critic, 선택/Phase 3)        │
│    - 결과를 qwen3-vl-abliterated:8b로 검사       │
│    - "손가락 6개", "구도 어긋남" → 자동 재생성   │
└──────────────────────────────────────────────┘
        │
        ▼
[결과 이미지] → 대화로 피드백 수신 → ①로 순환
```

## 두뇌 하이브리드 라우팅 규칙 (초안)

| 작업 유형 | 라우팅 | 이유 |
|-----------|--------|------|
| 일상 대화, 간단한 프롬프트 정리 | 로컬 (ollama) | 빠름, 무료, 프라이버시 |
| NSFW 프롬프트 합성 | **로컬 (무검열 모델 필수)** | 클라우드 API는 거부함 |
| 복잡한 의도 해석 / 다단계 추론 | 클라우드 API | 품질 우위 |
| 레퍼런스 이미지 분석 | 로컬 qwen3-vl-abliterated:8b (1차) → 애매하면 API | 프라이버시 + 품질 + NSFW 거부 없음 |
| 자기평가(Critic) | 로컬 qwen3-vl-abliterated:8b | 반복 호출이라 비용 부담 |

> 라우팅 기준은 "복잡도 + 검열 여부 + 비용"의 3축. 상세 임계값은 구현 단계에서 튜닝.

## 핵심 설계 원칙

1. **워크플로우는 동적 생성**: ComfyUI 노드 그래프를 두뇌가 상황에 맞춰 JSON으로 조립 → 제출. (고정 워크플로우 X). 단, 자주 쓰는 패턴은 **템플릿 그래프**로 두고 슬롯만 채우는 하이브리드 방식.
2. **VRAM 단일 점유 원칙**: LLM과 무거운 이미지 모델(Flux)은 동시에 VRAM에 올리지 않는다. → [02 문서](02-하드웨어-VRAM-오케스트레이션.md)
3. **모델 비종속 추상화**: 체크포인트가 Illustrious든 Flux든, 두뇌는 "intent"만 다루고 모델별 차이는 Prompt Compiler / Param Resolver가 흡수.
4. **점진적 자율성**: 처음엔 두뇌가 제안 → 사용자 확인. 신뢰 쌓이면 자동 실행 비율 증가.
