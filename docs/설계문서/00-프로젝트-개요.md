# 00. 프로젝트 개요

> 최종 수정: 2026-06-10

## 한 줄 정의

로컬 이미지 생성 모델(SDXL/Flux 계열) 위에 **대화형 에이전트 두뇌**를 얹어, GPT 이미지 생성처럼 "말로 대충 설명하거나 레퍼런스 이미지를 보여주는 것만으로" 고품질 이미지를 뽑아내는 **개인용 로컬 이미지 생성 AI**.

## 만드는 이유 (Problem)

기존 로컬 이미지 생성 도구(A1111, ComfyUI 등)의 진입장벽:

1. i2i / ControlNet / inpaint 등 작업 방식을 사용자가 직접 선택해야 함
2. 파라미터(steps, cfg, sampler, denoise 등)를 수동 튜닝해야 함
3. 프롬프트를 영어로, 모델별 정해진 포맷(태그/자연어)에 맞춰 직접 작성해야 함
4. 네거티브·퀄리티 토큰·가중치 등 노하우가 필요함

→ GPT 이미지 생성이 "마법처럼" 느껴지는 이유는 **모델이 아니라 모델 앞단의 오케스트레이션** 때문. 이 프로젝트의 본질은 **새 이미지 모델을 만드는 게 아니라, 기존 로컬 모델 위에 의도해석·라우팅·프롬프트합성·자기평가 두뇌를 얹는 것.**

## 목표 (Goals)

- **저점 보장**: 사용자가 한국어로 개떡같이 말해도 "쓸 만한" 결과가 나온다.
- **고점 근접**: 잘 쓰면 수동 ComfyUI에 근접하는 품질까지 끌어올릴 수 있다.
- **대화형 UX**: 채팅창에서 자연어 + 레퍼런스 이미지로 작업. 업계(Gemini/Claude/GPT) 채팅 UX를 레퍼런스로.
- **로컬 + 개인 전용**: NSFW 포함 무검열. 데이터 외부 유출 없음.
- **리소스 효율**: 16GB VRAM 안에서 LLM과 이미지 모델이 충돌하지 않도록 동적 로드/언로드.

## 비목표 (Non-Goals)

- 이미지 생성 모델 자체를 새로 학습/파인튜닝하는 것 (기존 체크포인트 활용)
- 다중 사용자 서비스 / 클라우드 배포 (개인 단일 사용자 전제)
- 클라우드 이미지 API 의존 (로컬 우선. 두뇌 LLM만 하이브리드 허용)

## 핵심 제약 (Constraints)

| 항목 | 값 |
|------|-----|
| GPU | RTX 4070 Ti SUPER (VRAM 16GB) |
| RAM | 64GB (모델 캐싱·CPU offload에 활용) |
| OS | Windows 11 |
| 두뇌 LLM | 하이브리드 (간단/무검열 → 로컬 ollama, 복잡/이미지분석 → 클라우드 API) |
| 도메인 | 애니/일러스트 중심 + 범용(포토리얼 포함) |
| 검열 | 없음 (로컬 개인용) |
