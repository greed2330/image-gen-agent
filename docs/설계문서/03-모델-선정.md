# 03. 모델 선정

> 최종 수정: 2026-06-10
> ⚠️ 이미지 생성 바닥은 한 달 단위로 SOTA가 바뀜. 아래는 2026-06 웹 조사 기준. 설치 직전 Civitai에서 최신 버전 재확인 권장.

## 큰 그림

"Stable Diffusion"은 단일 제품이 아니라 **아키텍처 계열**. 품질은 "SD냐"가 아니라 **어떤 파인튜닝 체크포인트냐**로 결정됨. 본 프로젝트는 단일 모델 고정이 아니라 **두뇌가 상황에 따라 체크포인트를 라우팅**하는 구조.

## 이미지 생성 모델

### 메인: 애니/일러스트 (SDXL 파생)

| 모델 | 특징 | VRAM | NSFW | 우선순위 |
|------|------|------|------|---------|
| **WAI-illustrious-SDXL** | Illustrious 파인튜닝 중 가장 대중적. 캐릭터 지식 방대, 자주 업데이트. 데일리 드라이버 추천 | ~8GB | 강력 | ★ 1순위 |
| **NoobAI-XL** | Illustrious 기반 추가 파인튜닝. danbooru 태그 관리 정교 | ~8GB | 강력 | ★ 1순위(대안) |
| **Pony Diffusion V6 XL** | 표현 범위 넓음, 범용성 | ~8GB | 강력 | 2순위(범용 보조) |
| Hassaku XL / Nova Anime XL | 스타일 중립 베이스 옵션 | ~8GB | 강력 | 보조 |

→ danbooru 태그 기반이라 **Prompt Compiler가 "태그 조립" 방식으로 깔끔하게 합성** 가능. LoRA/ControlNet/IPAdapter 생태계 최대 → "포즈 변경/레퍼런스 반영" 기능에 최적.

### 보조: 포토리얼 / 고점

| 모델 | 특징 | VRAM | NSFW | 비고 |
|------|------|------|------|------|
| **Chroma** (포토리얼 NSFW 메인) | FLUX.1-schnell 기반 8.9B, **완전 무검열**(해부학 재학습, content filter 0). 애니/실사/아트 5M 데이터 학습 | fp8 ~13GB / GGUF Q4~Q8 더 작음 | ✅ **무검열** | **포토리얼 NSFW = 이것** |
| ~~바닐라 Flux.1 dev~~ (사용 안 함) | 베이스가 **약하게 검열됨, NSFW 비권장** + **비상업 라이선스**(개인용만). 굳이 쓸 이유 없음 | — | ❌ | NSFW엔 부적합 → 제외 |
| SDXL 리얼리스틱 머지 | 가벼운 포토리얼 | ~8GB | 머지에 따라 | 빠른 실사 보조 |

> **라이선스/검열 확정 (2026-06 검증):**
> - WAI/NoobAI: Civitai 무료, danbooru 학습 → **무료·무검열**.
> - **Chroma: Apache 2.0 라이선스 + 완전 무검열 + 무료.** 포토리얼 NSFW는 Chroma로 통일. (바닐라 Flux 아님)
> - 바닐라 Flux.1 dev는 검열적 + 비상업 라이선스라 **사용하지 않음**. Chroma가 모든 면에서 우월.
> Chroma는 fp8 ~13GB라 **LLM과 동시 점유 불가** → 시분할 필수 ([02 문서](02-하드웨어-VRAM-오케스트레이션.md)). GGUF 양자화로 더 줄일 수 있음.

### 보조 도구 모델

- **ControlNet** (포즈/깊이/캐니): 레퍼런스의 "구도/포즈만 가져오기"
- **IPAdapter**: 레퍼런스의 "스타일/인물 특징만 가져오기"
- **VAE / 업스케일러(예: 4x-UltraSharp)**: 후처리 품질

### 프롬프트 합성 전용 모델 (TIPO / DanTagGen) ★ 2026-06-10 추가

| 모델 | 정체 | 크기 | 역할 |
|------|------|------|------|
| **TIPO** (KGen) | danbooru 태그+자연어 전용 학습 프롬프트 생성기. ICLR 2026 발표 | 200M / 500M | 거친 씨앗 태그 → 완성 danbooru 프롬프트로 확장·정렬 |
| DanTagGen (DTG) | TIPO 전신, 태그 전용 (400M, danbooru 상위 500만) | 400M | 경량 대안. TIPO로 충분하면 불필요 |

> **도입 이유 (오너 제안)**: "이미지 생성 프롬프트 합성 스페셜리스트가 따로 있나?" → 있음.
> - **이미지 모델(Illustrious/NoobAI)과 TIPO가 둘 다 danbooru 태그로 학습 → 같은 공용어.** 별도 포맷 정의 불필요(검증된 사실).
> - 범용 LLM의 어휘 부담을 덜어 hallucination 구조적으로 감소. allowlist 검증과 합쳐 사실상 해결.
> - ComfyUI 노드 존재 → "ComfyUI=엔진" 구조에 직결.
>
> **✅ 실측 완료 (2026-06-18, Phase 1)**: NSFW 태그 품질 검증됨 — 무검열, danbooru 관례 일치. **구현 결정**: `tipo-kgen`(pip) + **transformers 백엔드**(GGUF/llama-cpp는 Windows DLL 문제로 기각) + **TIPO-500M-ft** 모델 + **GPU 구동**(CPU long 82s 사용불가, GPU 1.2s). backend venv(`backend/.venv`, torch 2.6.0+cu124) 설치. ComfyUI 노드(z-tipo)는 두뇌의 seed→TIPO→allowlist 분리·트레이스 가시성 깨져서 기각. ⚠️ 검열태그 자동주입 이슈는 [07 §2 실측 발견] 참조.

## 두뇌 LLM (하이브리드)

### 로컬 (ollama)

> 오너 확정 모델 배정 (2026-06-10 결정). 무검열 모델 2종 추가로 NSFW 전 파이프라인 커버.

| 역할 | 배정 모델 | 크기 | 상태 |
|------|-----------|------|------|
| 메인 대화·의도해석 (SFW) | `qwen3:14b` | 9.3GB | 보유 ✅ |
| 빠른 대화·경량 워커 | `qwen3:4b` | 2.5GB | 보유 ✅ (SDXL과 공존 가능) |
| **프롬프트 합성 (NSFW)** | **`huihui_ai/qwen3-abliterated:14b`** | ~9GB | pull 예정 ⬇️ |
| **레퍼런스 분석 / Critic (비전, SFW+NSFW)** | **`huihui_ai/qwen3-vl-abliterated:8b`** | 6.1GB | pull 후 교체 ⬇️ (`qwen3-vl:8b` 대체) |
| mem0 / RAG 임베딩 | `nomic-embed-text` | 274MB | 보유 ✅ |
| JSON 구조화 출력 (SFW 합성 보조) | `qwen2.5-coder:14b` | 9.0GB | 보유 ✅ |
| 예비 | `gemma4:12b` | 7.6GB | 보유 (qwen3:14b와 역할 중복) |

> **비전 모델 교체 이유**: `qwen3-vl:8b`(검열)는 NSFW 생성물을 Critic 루프에서 분석 요청 시 거부 가능.
> `qwen3-vl-abliterated:8b`로 교체하면 동일 크기(6.1GB)로 NSFW Critic까지 커버.
> pull 완료 후 `ollama rm qwen3-vl:8b`로 기존 모델 제거 권장 (디스크 절약).

### 클라우드 API (보조)

- 복잡한 의도 해석 / 다단계 추론 / 애매한 레퍼런스 해석에만 호출
- 비용 최소화: 이미지당 텍스트 토큰만 소비 → 매우 저렴
- 주의: 클라우드 API는 NSFW 관련 처리 거부 가능 → 검열 무관 작업만 라우팅

## 설치 우선순위 (1차)

1. ComfyUI 설치 (오너 미보유 — 신규 설치) + WAI-illustrious(or NoobAI) 체크포인트
2. ControlNet + IPAdapter (SDXL용)
3. ollama: **무검열 텍스트 모델 1개만 추가** pull (비전 `qwen3-vl:8b`·임베딩 `nomic-embed-text`는 이미 보유)
4. (선택) Chroma — 포토리얼 NSFW가 필요해지면 (바닐라 Flux 아님)
