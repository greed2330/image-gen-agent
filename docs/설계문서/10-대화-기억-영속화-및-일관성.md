## 문제 정의 — 실측 확정

1. **영속 저장 전무.** `routes_chat.py:12`가 `_chats: dict = {}` (프로세스 메모리). 프론트엔드는 React 상태. `*.db`/`backend/data` 부재. → 재시작·새로고침 시 전부 소실.
2. **캐릭터 일관성 시소.** 정체성↔의상이 번갈아 사라짐. history 롤링 6개 윈도우라 길어지면 초반 정의 밀려남.
3. **생성 기록이 어디에도 안 남음.** 프롬프트·파라미터·seed·이미지·소요시간이 휘발 → 갤러리·재현·검색 불가.

## 저장 대상 (이게 한두 개가 아님 — 관계형 구조)

```
채팅방(room) ──< 메시지(message)
       └────────< 생성기록(generation)   [params, seed, prompt, image_path, trace]
캐릭터 카드 = room에 부속(누적 identity_tags)
```

## 저장소 결정 — SQLite + SQLModel (왜 / 버린 대안)

- **왜 SQLite:** 로컬 단일 파일, 서버 불필요 → "완전 로컬" 원칙 부합. 생성 기록이 **하루 수십~수백 건**(오너 사용 패턴) 쌓이면 곧 수천 건 → "전체 방 최근순 갤러리", 태그·파라미터 검색, "이 설정으로 재현"이 관계형 쿼리·인덱스로 깔끔. 데이터가 명백히 관계형.
- **버린 대안 — 방별 JSON 파일:** 채팅방까진 OK이나 교차-방 쿼리/검색이 전체 스캔이라 건수 늘면 느려짐. (이전 doc 10 초안이 이 방식이었고, 생성 기록을 못 담아 폐기.)
- **왜 SQLModel(ORM):** 이미 Pydantic 생태계 → 스키마·DB 모델 통합, 보일러플레이트 제거, FastAPI 통합. **그리고 배포 대비 이식성**: SQLite ↔ PostgreSQL을 연결 문자열만 바꿔 이전 가능(모델/쿼리 불변). raw `sqlite3`는 이전 시 쿼리 재작성 필요 → 비쌈. (라이브러리 도입 근거: 메모리 규칙 충족.)
- **배포용 DB는 추후 결정(오너, 2026-06-18):** 현재는 로컬 데스크톱 성격(로컬 GPU/모델/NSFW)이라 SQLite가 영구적으로 적합. 만약 중앙 다중사용자 SaaS로 가면 PostgreSQL로 이전(SQLModel 덕에 저비용). **지금은 SQLite로 빠르게 진행.**

> **mem0(doc 04)와의 관계:** SQLite는 방별 영속·정체성·생성기록(당장 필요분). mem0/Qdrant는 방-넘는 전역 취향·의미 검색용으로 **나중에**. 이번엔 mem0 구현 안 함.

## B. DB 테이블 — `backend/app/models/db.py` (신규)

> API I/O용 `schemas.py`(Intent/GenResult 등)와 분리. DB 테이블만 여기.

```python
import time, uuid
from typing import Optional
from sqlmodel import SQLModel, Field, Column, JSON


def _uuid() -> str:
    return str(uuid.uuid4())


class Room(SQLModel, table=True):
    id: str = Field(default_factory=_uuid, primary_key=True)
    name: str = "새 채팅"
    created_at: float = Field(default_factory=time.time)
    updated_at: float = Field(default_factory=time.time)
    nsfw_default: int = 0
    identity_tags: list[str] = Field(default_factory=list, sa_column=Column(JSON))  # 누적 캐릭터 카드


class Message(SQLModel, table=True):
    id: str = Field(default_factory=_uuid, primary_key=True)
    room_id: str = Field(index=True, foreign_key="room.id")
    role: str                       # "user" | "ai"
    text: Optional[str] = None
    image_path: Optional[str] = None
    reference_image: Optional[str] = None        # 첨부 입력 이미지(파일명)
    generation_id: Optional[str] = Field(default=None, foreign_key="generation.id")
    created_at: float = Field(default_factory=time.time)


class Generation(SQLModel, table=True):
    id: str = Field(default_factory=_uuid, primary_key=True)
    room_id: str = Field(index=True, foreign_key="room.id")
    created_at: float = Field(default_factory=time.time, index=True)
    # 요청
    user_message: str = ""
    reference_image: Optional[str] = None
    # 의도
    identity_tags: list[str] = Field(default_factory=list, sa_column=Column(JSON))
    scene_tags: list[str] = Field(default_factory=list, sa_column=Column(JSON))
    nsfw_level: int = 0
    style: Optional[str] = None
    # 라우팅
    workflow: str = "txt2img"
    checkpoint: str = ""
    model_profile: str = ""
    # 컴파일된 프롬프트
    positive: list[str] = Field(default_factory=list, sa_column=Column(JSON))
    negative: list[str] = Field(default_factory=list, sa_column=Column(JSON))
    # 파라미터 (재현용 — seed 필수)
    steps: int = 0
    cfg: float = 0.0
    sampler: str = ""
    scheduler: str = ""
    width: int = 0
    height: int = 0
    denoise: float = 1.0
    seed: int = 0
    # 결과
    image_path: Optional[str] = None
    duration_ms: float = 0.0
    status: str = "ok"              # "ok" | "error"
    error: Optional[str] = None
```

## C. DB 엔진 — `backend/app/db.py` (신규)

```python
from pathlib import Path
from sqlmodel import SQLModel, create_engine, Session

_DB_PATH = Path(__file__).parent.parent / "data" / "app.db"
_engine = None


def init_db() -> None:
    global _engine
    _DB_PATH.parent.mkdir(parents=True, exist_ok=True)
    _engine = create_engine(f"sqlite:///{_DB_PATH}", connect_args={"check_same_thread": False})
    import app.models.db  # noqa: F401  (테이블 등록)
    SQLModel.metadata.create_all(_engine)


def get_session() -> Session:
    return Session(_engine)
```

## D. 저장소 서비스 — `backend/app/services/chat_store.py` (신규)

방·메시지·카드 CRUD + 생성기록 저장. 단일 사용자·저빈도라 세션 단순 사용(락 불필요).

```python
from app.db import get_session
from app.models.db import Room, Message, Generation
from sqlmodel import select
import time

class ChatStore:
    # --- 방 ---
    def create_room(self, name=None) -> Room: ...           # Room 생성·저장·반환
    def get_room(self, room_id) -> Room | None: ...
    def list_rooms(self) -> list[Room]: ...                 # updated_at desc
    def update_room(self, room_id, *, name=None) -> Room | None: ...
    def delete_room(self, room_id) -> bool: ...             # 방+메시지+생성기록 cascade 삭제

    # --- 메시지 ---
    def add_message(self, room_id, role, *, text=None, image_path=None,
                    reference_image=None, generation_id=None) -> Message: ...
    def list_messages(self, room_id) -> list[Message]: ...  # created_at asc

    # --- 캐릭터 카드 ---
    def get_identity(self, room_id) -> list[str]:
        r = self.get_room(room_id); return r.identity_tags if r else []
    def set_identity(self, room_id, tags: list[str]) -> None: ...   # room.identity_tags 갱신

    # --- 생성기록 ---
    def add_generation(self, gen: Generation) -> Generation: ...
    def list_generations(self, room_id=None, limit=50) -> list[Generation]: ...  # created_at desc
```

> 각 메서드는 `with get_session() as s:` 패턴으로 구현. 방 삭제 시 메시지·생성기록도 함께 삭제(수동 delete 또는 FK cascade).

## E. 캐릭터 카드 누적 — `intent_parser.py`

`IntentParser`에 `store: ChatStore` 주입. 속성군 배타 병합(doc 09 `tag_attribute_groups.yaml` 공유 — 로딩 헬퍼 1곳에서):

```python
def _merge_identity(card_tags, new_tags, groups) -> list[str]:
    result = [_normalize_tag(t) for t in card_tags]
    for raw in new_tags:
        nt = _normalize_tag(raw)
        g = _attr_group_of(nt, groups)
        if g:
            result = [t for t in result if _attr_group_of(t, groups) != g]  # 같은 군 교체
        if nt not in result:
            result.append(nt)
    return result
```

`parse()` 말미:

```python
prev = self._store.get_identity(request.chat_id)
merged = _merge_identity(prev, intent.identity_tags, self._groups)
intent.identity_tags = merged                     # 이번 생성에 누적 정체성 주입
self._store.set_identity(request.chat_id, merged)
```

→ history 윈도우와 무관하게 방 누적 정체성이 매 생성에 주입됨. 정체성↔의상 시소 해소.

## F. 생성기록 저장 + seed 캡처 — `orchestrator.py`

**seed는 현재 `_build_workflow` 내부에서 random 생성 후 버려진다.** 재현하려면 캡처 필요:

1. `_build_workflow`가 seed를 내부 생성하지 말고 **인자로 받도록** 변경(`def _build_workflow(compiled, params, route, seed, input_image=None)`).
2. `run()` ⑤에서 `seed = random.randint(0, 2**31-1)` 생성 → 빌더에 전달.
3. ⑥ 이후(또는 finally 후) `Generation` 레코드 조립·저장:

```python
gen = Generation(
    room_id=request.chat_id, user_message=request.message,
    reference_image=intent.reference,
    identity_tags=intent.identity_tags, scene_tags=intent.scene_tags,
    nsfw_level=intent.nsfw_level.value, style=intent.style,
    workflow=route.workflow.value, checkpoint=route.checkpoint,
    model_profile=route.model_profile.value,
    positive=compiled.positive, negative=compiled.negative,
    steps=params.steps, cfg=params.cfg, sampler=params.sampler,
    scheduler=params.scheduler, width=params.resolution.width,
    height=params.resolution.height, denoise=params.denoise, seed=seed,
    image_path=image_path, duration_ms=<총 소요>, status="ok",
)
self._store.add_generation(gen)
```

실패 시 `status="error", error=str(exc)`로 저장(except 경로). `Orchestrator.__init__`에 `store: ChatStore` 추가.

4. `GenResult`에 `seed: Optional[int] = None`, `generation_id: Optional[str] = None` 추가(프론트 재현·표시용).

## G. 라우트 — `routes_chat.py` 전면 교체 + 생성기록 조회

```
POST   /chats              {name?}            → Room
GET    /chats                                 → [Room] (목록)
GET    /chats/{id}                            → {room, messages}  (전체 복원)
PATCH  /chats/{id}         {name?}            → Room
DELETE /chats/{id}                            → {deleted}
POST   /chats/{id}/messages {role,text,...}   → Message (메시지 append)
GET    /generations?room_id=&limit=           → [Generation]  (갤러리/검색)
```

`POST /chats/{id}/generate`는 유지: 생성 후 orchestrator가 Generation 저장 + AI 메시지(`image_path`, `generation_id`)를 `add_message`로 append.
