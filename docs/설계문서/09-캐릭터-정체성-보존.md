## 문제 정의 — 실측으로 확정 (추측 아님)

증상: "17-18세 백호 수인 소녀 + 스포츠브라/스패츠" 요청 시 **수인 정체성이 통째로 사라지고 평범한 인간 소녀**로 나옴(테스트 2번 이미지: 검은 머리, 호랑이 귀·꼬리·흰털 전부 없음).

`POST /generate/dry-run`으로 단계별 trace를 실측한 결과(2026-06-18):

**① IntentParser — 정상.** 입력에서 정체성 태그를 제대로 뽑음:
```
seed_tags: 1girl, solo, teen, sports bra, spats,
           animal ears, tiger ears, tiger tail, white hair, kemonomimi
```

**③ TIPO 확장 — 여기가 범인.** 10개 seed → **40개**로 부풀리며 두 가지 사고:

1. **직접 모순 태그 주입:** `white hair`가 있는데 `black hair`, `multicolored hair`, `streaked hair`를 같이 박음 → 머리색 충돌 → **생성 시 검은 머리 당첨**.
2. **정체성 익사:** 수인 정체성 태그 ~10개 vs TIPO가 추가한 인간 몸/의상 태그(`navel, collarbone, thighhighs, medium breasts, cleavage, midriff, black shorts, bare shoulders, black sports bra, stomach`...) ~23개. **소수가 다수에 압도**되어 모델이 "운동복 입은 인간 소녀"로 그림.

```
정체성 ~10  vs  인간몸/의상 ~23 + 머리색 모순 3
        → 모델이 다수+모순을 따라감 → 수인 증발
```

> **이전 오진 폐기:** "흰색→주황 색 드리프트"는 헛다리였다. 흰색은 실제로 정상 렌더됨(테스트 1·3번). 실제 모순은 `black hair`/`multicolored hair`였고, 더 큰 문제는 **TIPO 과확장에 의한 정체성 익사**다.
>
> **프리셋 안 폐기 이유:** 프리셋(수동 태그 고정)도 결국 `seed_tags`로 들어가 똑같이 TIPO에 익사한다. 매번 수동 관리 비용까지 든다. → 합성 파이프라인 자체를 고친다. (오너 지적, 2026-06-18)

## 핵심 원리 — WHO를 잠그고 WHAT만 확장

태그를 두 종류로 나눈다:

- **정체성(WHO)** — 종족, 털/몸·머리·눈 색, 머리 스타일, 나이, 영구 특징. → **TIPO에 안 넣는다. 강조 가중치로 살린다. 모순 태그는 차단한다.**
- **장면(WHAT)** — 포즈, 동작, 표정, 이번 요청 의상, 배경. → TIPO로 자유 확장.

세 가지 방어 장치:

1. **강조(emphasis):** 정체성 태그를 `(tiger ears:1.3)` 형태로 가중 → 다수에 안 묻힘.
2. **속성군 배타(category exclusion):** 정체성이 한 속성군(머리색 등)의 값을 점유하면, TIPO가 같은 군의 **다른 값**을 뱉어도 positive에서 빼고 negative로 보냄 (white hair 점유 → black hair·multicolored hair 차단).
3. **TIPO 범위 제한:** 정체성은 TIPO 입력에서 제외하고 장면만 확장 → 익사 원천 차단.

## 1-C. `prompt_compiler.py` — 헬퍼 + compile() 재작성

상단 헬퍼 추가:

```python
def _normalize_tag(tag: str) -> str:
    return tag.strip().lower().replace("_", " ")


def _attr_group_of(tag: str, groups: dict[str, list[str]]) -> str | None:
    for gname, members in groups.items():
        if tag in members:
            return gname
    return None
```

경로 상수 + 로딩:

```python
_GROUPS_PATH = Path(__file__).parent.parent / "presets" / "tag_attribute_groups.yaml"

# __init__:
self._groups: dict[str, list[str]] = {}

# load_presets() 끝에 추가:
with _GROUPS_PATH.open(encoding="utf-8") as f:
    raw = yaml.safe_load(f) or {}
self._groups = {g: [_normalize_tag(t) for t in members] for g, members in raw.items()}
```

`compile()`의 SDXL 경로(Chroma 분기 이후)를 아래로 교체:

```python
expanded = await self._tipo.expand_tags(
    intent.seed_tags, route.model_profile.value, nsfw_level=intent.nsfw_level.value
)

# 보호 대상 = 유저가 말한 seed_tags. TIPO에 익사시키지 않고 강조로 재주장.
protected = [_normalize_tag(t) for t in intent.seed_tags if t.strip()]
protected_set = set(protected)

# 정체성이 점유한 속성군 값 수집 (머리색·눈색 등)
claimed: dict[str, set[str]] = {}
for t in protected:
    g = _attr_group_of(t, self._groups)
    if g:
        claimed.setdefault(g, set()).add(t)

# TIPO 확장분 분류: 검열/중복/모순 제거
censored_found = [t for t in expanded if t in CENSORED_TAGS]
extras_raw: list[str] = []
conflict_neg: list[str] = []
for t in expanded:
    if t in CENSORED_TAGS:
        continue
    nt = _normalize_tag(t)
    if nt in protected_set:
        continue  # 이미 보호 태그
    g = _attr_group_of(nt, self._groups)
    if g and g in claimed and nt not in claimed[g]:
        conflict_neg.append(nt)   # 정체성과 모순 → negative
        continue
    extras_raw.append(t)

valid_extras, dropped = self._allowlist.validate(extras_raw)

key = _PROFILE_KEY[route.model_profile]
preset = self._presets.get(key, {})
weight = preset.get("identity_emphasis", 1.3)
emphasized = [f"({t}:{weight})" for t in protected]

quality_pos: list[str] = preset.get("quality_pos", [])
quality_neg: list[str] = preset.get("quality_neg", [])
forced_neg = list(CENSORED_TAGS)
negative = _dedup(quality_neg + conflict_neg + censored_found + forced_neg)

logger.info(
    "compile: %d protected(emphasis %.2f) + %d extras (dropped %d, conflict→neg %d)",
    len(emphasized), weight, len(valid_extras), len(dropped), len(conflict_neg),
)
return CompiledPrompt(
    positive=quality_pos + emphasized + valid_extras,
    negative=negative,
    model_profile=route.model_profile,
    dropped_tags=dropped,
)
```

# 2단계 — 슬롯 분리 (정체성/장면) + TIPO 장면 전용

> 1단계 검증 통과 후. 1단계 메커니즘을 재활용하되 보호 대상을 `identity_tags`로 한정하고, **TIPO를 장면 태그에만** 적용한다. 이 구조가 기억 영속화(doc 10: 캐릭터 카드 = 저장된 identity_tags)의 토대다.

## 2-A. `schemas.py` — `Intent` 변경

```python
class Intent(BaseModel):
    subjects: list[str] = Field(default_factory=list)
    style: Optional[str] = None
    setting: Optional[str] = None
    mood: Optional[str] = None
    nsfw_level: NsfwLevel = NsfwLevel.SAFE
    reference: Optional[str] = None
    workflow_hint: Optional[WorkflowType] = None
    identity_tags: list[str] = Field(default_factory=list)   # WHO — TIPO 제외·강조·보호
    scene_tags: list[str] = Field(default_factory=list)      # WHAT — TIPO 확장

    @property
    def seed_tags(self) -> list[str]:
        return self.identity_tags + self.scene_tags
```

## 2-B. `intent_parser.py` — 2-배열 출력

`_SYSTEM_PROMPT`를 `seed_tags` 단일 → `identity_tags` + `scene_tags`로 교체. 모든 few-shot도 두 배열로. 예:

```
Input: "백호 수인 소녀, 스포츠 브라 입고 브이"
Output: {"subjects":["1girl"],"style":"anime","setting":null,"mood":null,"nsfw_level":1,"identity_tags":["1girl","solo","animal ears","tiger ears","tiger tail","white hair","white fur","white body","kemonomimi","amber eyes"],"scene_tags":["sports bra","v sign","midriff"],"workflow_hint":null}
```

규칙 추가:

```
- identity_tags: species, fur/body color, hair, eyes, age, permanent features, AND clothing the user says the character "always wears". 6-10 tags.
- scene_tags: pose, action, expression, THIS-request clothing, background, mood. Clothing defaults here; promote to identity only if "always wears".
- Carry over identity from prior turns into identity_tags; put only the new request's changes into scene_tags.
```

`parse()`에서 두 필드 읽기(`identity_tags`, `scene_tags`).

## 2-C. `prompt_compiler.py` — TIPO 장면 전용 + 보호 대상 변경

```python
# TIPO 입력: 주체 + 장면만 (정체성 외형은 제외해 익사·모순 원천 차단)
tipo_input = intent.subjects + intent.scene_tags
expanded = await self._tipo.expand_tags(
    tipo_input, route.model_profile.value, nsfw_level=intent.nsfw_level.value
)

# 보호 대상 = identity_tags (1단계의 protected 소스 교체)
protected = [_normalize_tag(t) for t in intent.identity_tags if t.strip()]
```

나머지(claimed 수집, 모순 배타, 강조, negative 조립)는 1단계와 동일.

# 3단계 — danbooru 단어장 최근접 보정 (필요 시)

1·2단계 후에도 확장 태그가 자주 드롭되면 착수.

`tag_allowlist.py validate()`에 드롭 전 보정:

```python
import difflib
# tag not in self._tags 일 때:
norm = tag.strip().lower().replace(" ", "_")     # danbooru CSV는 보통 언더스코어
if norm in self._tags:
    valid.append(norm); continue
near = difflib.get_close_matches(norm, self._tags_list, n=1, cutoff=0.92)
if near:
    valid.append(near[0]); continue
dropped.append(tag)
```

`load()`에서 `self._tags_list = list(self._tags)` 보관.

> **전제:** `danbooru_tags.csv` 미보유 상태(기동 로그 "allowlist disabled" 확인됨). 3단계 착수 시 CSV부터 확보(a1111-sd-webui-tagcomplete). 현재 allowlist가 꺼져 있어 1·2단계는 allowlist 없이도 동작하나, **확장 태그 검증 안전망이 없는 상태**임을 유의.
