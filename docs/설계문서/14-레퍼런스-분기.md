# 14. 레퍼런스 분기 (IPAdapter / ControlNet / img2img)

> 상태: 설계 (2026-06-24). 구현 전 오너 확인 필요 항목은 §7 참조.
> 관련: Doc 11(img2img), 체크리스트 「Phase 3」·「알려진 한계」.

---

## 1. 목표

채팅 + 레퍼런스 이미지로 **"이 캐릭터로 / 이 포즈로 / 이런 느낌으로"** 를 자동 분기해 처리한다.

- **현재 한계:** 레퍼런스가 있으면 **무조건 img2img(통째 변형)**. → 캐릭터만 옮기거나 포즈만 옮기는 걸 못 함.
- **이번 작업:** 레퍼런스 활용 의도를 3가지로 갈라 각각 다른 ComfyUI 워크플로로 보낸다.

---

## 2. 핵심 개념 — "레퍼런스에서 **무엇을 KEEP** 하나"

레퍼런스 1장에서 가져올 수 있는 건 ① 캐릭터 외형, ② 포즈/구도, ③ 그림체. 분기는 무엇을 유지하느냐로 갈린다.

| reference_mode | 레퍼런스에서 KEEP | 텍스트로 CHANGE | 사용자 표현 예 |
|---|---|---|---|
| **character** (IPAdapter) | 캐릭터 외형·정체성·분위기 | 포즈·배경·행동 | "이 **캐릭터**를 ~하게" |
| **pose** (ControlNet) | 포즈·골격·구도 | 캐릭터 외형·색 | "이 **포즈/자세**로 ~한 캐릭터를" |
| **vary** (img2img) | 전체 통째(구도+색+외형) | 약한 변형 | "이 이미지를 ~느낌으로 살짝" |

> character와 pose는 **거울쌍**이다: 무엇을 유지하고 무엇을 바꾸는지가 정반대.
> "그림체 그대로"는 대개 **동일 체크포인트**가 처리하므로 별도 스타일 이식이 꼭 필요하진 않다.

---

## 3. 분기 결정 — 명시적 모드 (UI 드롭다운)

**분기는 사용자가 채팅 위 드롭다운에서 직접 고른다** (오너 제안, 2026-06-24). `reference_mode`(character/pose/vary)는 **프론트가 생성 요청에 실어 보내며, LLM이 분류하지 않는다.**

### 모드 (레퍼런스 첨부 시 활성)
| 모드 | KEEP | 라벨 / 툴팁 |
|---|---|---|
| **character** | 캐릭터 외형 | "캐릭터 이식 — 이 캐릭터를 새 포즈·장면으로" |
| **pose** | 포즈·골격 | "포즈 이식 — 이 자세 그대로 다른 캐릭터를" |
| **vary** | 전체 | "변형 — 이 이미지를 살짝 바꿔" |

- 레퍼런스 없으면 모드 무의미 → 일반 txt2img (드롭다운 비활성/숨김).
- 기본값 **vary**: 미선택·애매할 때 원본 보존이 가장 안전.

### 왜 명시적 모드인가 (자동분류 A·비전 B 모두 기각)
- **A(텍스트 자동분류) 기각:** 유일 약점이 "말투 오독 → 엉뚱한 분기"였는데, **명시 선택은 그 실패가 0**. 백엔드도 단순(LLM 분류 불필요).
- **B(qwen3-vl) 기각:** "무엇을 KEEP할지"는 이미지가 아니라 사용자 의도에 있음 + VRAM 스왑 부담. (qwen3-vl은 추후 "레퍼런스→태그 자동추출" 보강용으로만 등판 가능.)
- **부수 이점:** 모듈화(모드→워크플로 1:1로 깔끔히 분리), 발견성(드롭다운+툴팁이 그대로 가이드 역할).

> 모듈화 원칙: 모드는 **레지스트리**(모드 → {라벨, 툴팁, 워크플로 템플릿, 슬롯 채우기})로 등록. 새 모드(inpaint 등) 추가 = 레지스트리 항목 1개 + 워크플로 JSON 1개. 라우터/오케스트레이터는 모드 키로 디스패치만.

## 4. 기능 어프로치 (파이프라인 변경)

```
① 사용자: 모드 선택(드롭다운) + 이미지 업로드 + 메시지
② IntentParser: 기존 태그만 파싱 (reference_mode는 안 건드림 — UI가 정함)
③ WorkflowRouter: reference 있음 + reference_mode(요청에 실림) →
     character → WorkflowType.IPADAPTER
     pose      → WorkflowType.CONTROLNET
     vary      → WorkflowType.IMG2IMG (기존)
   reference 없음 → TXT2IMG (모드 무시)
④ orchestrator._build_workflow: 모드별 템플릿 (모드 → 워크플로 1:1)
     IPAdapter   → ipadapter.json   (레퍼런스 → IPAdapterModelLoader/Apply)
     ControlNet  → controlnet.json  (레퍼런스 → openpose 전처리 → ControlNet Apply)
     img2img     → img2img.json     (기존)
⑤ 생성 (기존 generate + 진행률 스트리밍 그대로)
```

### 코드 변경 지점
- `schemas.py`: `ReferenceMode` enum(character/pose/vary). `GenRequest.reference_mode`(**UI가 보냄**, 기본 vary). `WorkflowType`엔 이미 IPADAPTER/CONTROLNET 존재.
- `intent_parser.py`: **변경 없음** (reference_mode 분류 안 함).
- `workflow_router.py`: `request.reference_mode` + reference 유무로 3분기(현재는 무조건 IMG2IMG).
- `workflows/ipadapter.json`, `workflows/controlnet.json` 신규.
- `orchestrator._build_workflow`: 모드 레지스트리로 템플릿 디스패치 + 레퍼런스 슬롯 채우기.
- **프론트:** composer 위 모드 드롭다운(레퍼런스 첨부 시 활성), 각 모드 툴팁. 선택값을 `reference_mode`로 전송.
- 업로드는 기존 `comfyui.upload_image` 재사용.

## 5. ComfyUI 노드 현실 (2026-06-24 실측) — 설계 제약

| 자산 | 상태 |
|---|---|
| IPAdapter_plus 노드 (IPAdapter/Advanced/ModelLoader 등) | ✅ 설치됨 |
| ip-adapter-plus_sdxl_vit-h / plus-face | ✅ 설치됨 |
| CLIP-ViT-H | ✅ 설치됨 |
| ControlNet union promax 모델 | ✅ 설치됨 |
| **포즈 전처리기 (comfyui_controlnet_aux / DWPose / OpenPose)** | ❌ **미설치** |
| Canny (코어 노드) | ✅ 사용 가능 |

**핵심 제약:** ControlNet은 입력 이미지를 **제어맵(control map)으로 전처리**해야 한다.
- **canny(윤곽)**: 코어 `Canny` 노드로 전처리기 설치 없이 가능 → "윤곽/구도 유지" 수준의 pose 분기 가능.
- **openpose/DWPose(골격 포즈)**: 전용 전처리기(comfyui_controlnet_aux) **설치 선행 필요** → 의존성 추가 결정 사항.

→ **character(IPAdapter)와 vary(img2img)는 즉시 구현 가능.** pose는 전처리기 수준에 따라 갈림(§6).

---

## 6. 단계 제안 (오너 결정 2026-06-24: pose = openpose)

- **단계 1 — character(IPAdapter) + vary(img2img):** 신규 설치 없이 즉시 빌드. "이 캐릭터로"가 핵심 가치 → 우선.
- **단계 2 — pose(openpose):** `comfyui_controlnet_aux` 설치 후 ControlNet + openpose **골격** 분기.
  - **canny 기각:** 윤곽선이 원본 캐릭터의 머리·얼굴 실루엣까지 포함 → "캐릭터 교체 + 포즈 유지" 의도에서 원본 외형이 딸려옴. openpose는 골격만 추출해 의도와 정확히 일치.
  - **comfyui_controlnet_aux 도입 정당성:** ComfyUI 코어엔 골격 포즈 전처리기가 없음(canny만). DWPose/OpenPose 전처리는 이 노드팩이 사실상 표준·대체 불가. 단계 2 진입 시 설치.
