## 주요 설계 결정 기록

> **캐릭터 프리셋 폐기:** 프리셋도 TIPO에 익사 + 매번 수동 관리 비용 → 오너 거부. 대신 **정체성 보존 + 자동 캐릭터 카드**로 전환(Doc 09·10).

> **백호 색 오진 정정:** "흰색→주황 드리프트"는 헛다리(흰색 정상 렌더). 진짜 문제는 **TIPO가 정체성을 익사시키고 모순 태그(black/multicolored hair)를 주입**하는 것. dry-run trace 실측 확정. → identity_tags 강조 + 속성군 배타 + TIPO 범위 제한으로 해결.

> **img2img 단순화:** "레퍼런스가 있으면 무조건 img2img"로 처리. qwen3-vl로 의도 분석 후 IPAdapter/ControlNet 분기는 Phase 3 별도 설계.

> **백호 수정 실측 확정 + 의도해석 설정 버그 (2026-06-24, session_011):** 백호 정체성 보존(Doc 09)을 dry-run으로 실측 확정 — identity 강조 + 모순 색 negative 이동 정상 작동. **배경/시간대 오염 가설은 12개 프롬프트 재현으로 반증** → 새 속성군 추가 안 함(1회 관측으로 과설계 금지). 별건으로 의도해석 매 요청 폴백(~52초) 원인 진단: runtime chat_model이 VL 8B로 잘못 설정돼 **빈 출력** → qwen3:14b 복원 + 부적합 모델(embed/vl) 필터. 상세: `docs/기타/디버깅기록/2026-06-24-intent-빈출력-VL모델.md`.
