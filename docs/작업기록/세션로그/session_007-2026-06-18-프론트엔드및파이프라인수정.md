## 주요 결정 (무엇을·왜·버린 대안)

### 1. sampler 이름 수정: `euler_a` → `euler_ancestral`
- **원인**: ComfyUI API가 허용하는 sampler 이름은 `euler_ancestral`. `euler_a`는 A1111 관례.
- **확인 방법**: `GET /object_info/KSampler`로 유효 목록 조회
- **적용**: `model_presets.yaml` illustrious_base, noobai_base 모두 수정

### 2. 백엔드 실행은 반드시 venv Python으로
- **원인**: `python -m uvicorn` (글로벌 python)으로 실행하면 `kgen` 모듈을 못 찾음. kgen은 `backend/.venv`에만 설치되어 있음.
- **올바른 실행 명령**: `E:\image-gen-agent\backend\.venv\Scripts\python.exe -m uvicorn app.main:app --host 127.0.0.1 --port 8000`

### 3. Critic stub → passthrough
- Phase 3 전까지 `NotImplementedError` 대신 `Critique(passed=True)` 반환하도록 수정
- 이유: Critic은 Phase 3 기능인데 에러를 던져 전체 파이프라인 차단

### 4. 이미지 서빙 엔드포인트 추가: `GET /images/{filename}`
- ComfyUI 출력 경로(`E:\ComfyUI\output\`)의 이미지를 HTTP로 서빙
- 경로 탐색(path traversal) 방어 포함 (`/`, `\`, `..` 거부)
- CORS 설정: `http://localhost:3000` 허용

### 5. Next.js 프론트엔드 구현 (mockup v5 기반)
- `frontend/` — Next.js 16.2.9, TypeScript, App Router
- 3단 레이아웃: 좌측(채팅방 목록) + 중앙(채팅) + 우측(최근 생성 갤러리)
- GenCard: 로딩 진행 바 → 3D flip → 실제 이미지
- 라이트박스: 클릭 확대, ESC 닫기
- 4가지 테마: dark/light/slate/sand (CSS 변수)
- 실제 백엔드 API 연동 (`POST /generate`, `GET /images/{filename}`)

### 6. 레퍼런스 이미지 첨부 UI 구현 (+버튼)
- 파일 선택 → composer에 썸네일 프리뷰 + 라벨 표시
- 전송 시 user 말풍선에 첨부 이미지 표시
- 백엔드로 base64(`reference_image`)로 전달 (현재 무시됨, Phase 3에서 연결)
- **이전 상태**: +버튼이 아무것도 안 하는 stub → 오너 지적으로 수정

### 7. 대화 맥락(history) IntentParser에 주입
- **문제**: 2번째 요청("스포츠 브라로")에서 1번 요청에서 확립된 "백호 수인" 특성이 완전히 사라짐
- **원인**: 각 메시지가 독립적으로 파싱됨
- **해결**: `GenRequest`에 `history: list[HistoryMessage]` 필드 추가. 프론트엔드에서 현재 채팅방의 이전 메시지들을 함께 전송. `_build_messages()`에서 LLM messages 배열에 user 이전 턴들을 포함.
- 최근 6개 user 메시지만 포함 (AI 응답 제외 — 의도 히스토리만 필요)

### 8. IntentParser 시스템 프롬프트 개선
- **문제**: "백호 수인" → `white tiger`(동물) 태그로 번역 → 수인 소녀가 아닌 동물/쿠션 출력
- **수정**: 수인(kemonomimi) 번역 규칙 명시: `animal ears`, `{species} ears`, `{species} tail`, `kemonomimi` 사용. "wolf"가 아닌 "wolf ears" 형태.
- 예시 추가: 백호 수인, 늑대 수인, 스포츠 브라+스패츠
- nsfw_level 1 조건 명확화: 스포츠 브라/수영복/맨배 = suggestive

### 버그 / 품질 이슈
1. **백호(白虎) 색상 미반영**: `white hair` 태그가 있어도 TIPO 확장 후 노란/주황 호랑이 배색으로 나옴. TIPO의 danbooru 분포에서 "white tiger" 맥락 태그가 일반 호랑이 쪽으로 흐르는 것으로 추정. → seed_tags에 `white fur`, `white body` 등 색상 태그를 더 강하게 고정하는 방식 검토 필요.

2. **history 미영속**: 브라우저 새로고침 시 채팅방과 history가 초기화됨. 현재는 React 상태(in-memory)만. → localStorage 또는 백엔드 DB 저장 필요 (Phase 4 미완).

3. **GenCard 진행률 시뮬레이션**: 실제 ComfyUI 진행률(WebSocket)이 아닌 타이머 시뮬레이션. → Phase 4: WebSocket `/generate/progress` 구현 필요.
