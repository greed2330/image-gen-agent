## 주요 결정 (무엇을·왜·버린 대안)

### 1. CloudLLMClient — SDK 없이 httpx 직접 사용
- Anthropic Messages API를 httpx로 직접 호출. `anthropic` 패키지 추가 불필요.
- **왜**: 이미 httpx가 의존성에 있음. 불필요한 패키지 추가 제거(Simplicity First).
- **버린 대안**: anthropic SDK — 추가 설치 비용 있고 Phase 2에서 cloud LLM은 optional.

### 2. MemoryService — 지연 초기화 (lazy init)
- `_mem` 속성을 첫 실제 호출 시 초기화. 서버 시작 시 Qdrant가 미실행 상태여도 OK.
- **왜**: Qdrant는 선택적 의존성. Phase 3까지 memory 기능은 보조.

### 3. IntentParser — 임상적 핸드오프 + escalation 패턴
- qwen3:14b에게 "생성"이 아닌 "분류"로 프레이밍 → 검열 모델 거부율 낮춤.
- EXPLICIT 등급 또는 JSON 파싱 실패 시 abliterated로 폴백.
- 출력 계약: 어느 모델 담당이든 동일 seed_tags JSON → 다운스트림 무관.
- **doc 07 §1-A 원칙 1·2·3 그대로 구현.**

### 4. DI — deps.py 모듈 레벨 싱글턴
- 복잡한 DI 프레임워크 없이 module-level 인스턴스로 단순화.
- `init_all()` → FastAPI lifespan에서 1회 호출 (preset + allowlist 로딩).
- **버린 대안**: FastAPI Depends — Phase 2 규모에서 과설계.

### 5. TIPO thread executor
- `loop.run_in_executor(None, ...)` — TIPO GPU 추론(blocking)을 async loop 밖 thread에서 실행.
- kgen의 `load_model` 모듈 레벨 캐시 활용 → 첫 호출만 느리고 이후 빠름.
