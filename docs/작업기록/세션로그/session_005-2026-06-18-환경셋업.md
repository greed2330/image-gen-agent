## 주요 결정 (무엇을·왜·버린 대안)

### 1. ComfyUI = Git clone + 전용 venv (`E:\ComfyUI`)
- 프로젝트 repo 밖에 설치(중첩 repo 회피). 백엔드는 API `127.0.0.1:8188`로 접속.
- torch 2.6.0+cu124, ComfyUI v0.25.0. **버린 대안**: Portable(임베디드 파이썬 분리), Desktop(경로 불투명).

### 2. 메인 체크포인트 2종 — HF 미러(토큰 불필요)
- WAI `misri/waiNSFWIllustrious_v130`(6.94GB, 단일파일) + NoobAI `Laxhar/noobai-XL-1.1`(7.11GB).
- **왜 HF 미러**: Civitai는 토큰 필요. **왜 단일 파일**: ComfyUI 체크포인트는 단일 .safetensors 필요(diffusers 분할 형식 repo 다수라 검증 후 선별).

### 3. ControlNet/IPAdapter — vit-h 라인으로 통일
- ControlNet union promax(`xinsir`, 10+ 제어 1모델) + IPAdapter plus/plus-face(vit-h) + CLIP-ViT-H 인코더 + 커스텀노드 `ComfyUI_IPAdapter_plus`(cubiq).
- **왜 vit-h만**: CLIP vision 인코더 1개로 통일(bigG ~3.5GB 생략, Simplicity First). 전처리기 aux노드는 Phase 3(레퍼런스 분석)으로 연기.

### 4. TIPO — tipo-kgen + transformers + TIPO-500M-ft + **GPU** (핵심 결정)
- backend venv(`backend/.venv`)에 설치. 의존성 추가: `tipo-kgen`, `sentencepiece`(토크나이저), `transformers`, `torch 2.6.0+cu124`(GPU).
- **왜 transformers 백엔드**: GGUF/llama-cpp-python은 Windows DLL 문제(문서화된 이슈). **왜 GPU**: CPU는 long 82초로 사용불가, GPU 1.2초. **왜 500M-ft**: 품질, CPU/GPU 모두 감당.
- **버린 대안**: ComfyUI z-tipo 노드(두뇌의 seed→TIPO→allowlist 분리·트레이스 가시성 깨짐).

## 주요 발견 (구현에 영향)

1. **TIPO가 `censored`/`mosaic censoring` 태그 자동 주입** (danbooru 분포). 무검열 프로젝트인데 모자이크 생성됨 → prompt_compiler ③ allowlist에서 검열태그를 negative로 강제 이동 필요. (07 §2에 기록)
2. **TIPO는 GPU 필수** (CPU long 82s). COMPILING 단계 공존: 두뇌(9.6)+TIPO(1.25)+baseline(1.4)≈12.3GB < 16GB.
3. TIPO 무검열 + danbooru 관례 일치 확인 → C3(TIPO Plan B) 불필요.
