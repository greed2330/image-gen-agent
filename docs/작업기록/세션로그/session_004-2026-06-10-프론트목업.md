## 세션 목표

심심풀이 겸 프론트 디자인 방향 탐색 — "피그마 미리보기 수준"의 동작하는 UI 목업을 HTML 단일 파일로 제작. 버전을 거듭하며 디자인 방향을 좁힘. (실제 구현은 Next.js이며 이 HTML은 일회성 프로토타입)

---

## 주요 결정 (무엇을·왜·버린 대안)

### 1. 목업 버저닝 규칙 신설 (CLAUDE.md)
- 오너 요청: 테스트/목업 코드는 **여러 버전을 남기며** 작업.
- CLAUDE.md `Coding Rules > Prototypes / Mockups (versioned)` 추가: `mockup/`에 보관, **이전 버전 덮어쓰기 금지·`-vN` 증가**, 버전 간 변경점 명시, 실제 구현은 Next.js.

### 2. 디자인 방향 — 5개 버전으로 수렴
- **v1** (`index.html`): 기본 — 사이드바/채팅/설정/테마전환/약관 체크박스 데모. "양산형·휑함" 피드백.
- **v2** (`index-v2.html`): 오로라 배경·글래스·글로우·스프링 떡칠 → 오너 "너무 AI가 만든 것 같다" 피드백. **버린 방향.**
- **v3** (`index-v3.html`): 정반대로 절제(Linear/Vercel풍) — 중립 회색조·액센트 1색·헤어라인·평평. "기교가 안 보인다" 피드백.
- **v4** (`index-v4.html`): 절제 톤 유지 + 실제 프론트 기교 — 3D 플립 생성카드, 커버플로우 갤러리, 마우스 틸트, 마그네틱 버튼.
- **v5** (`index-v5.html`): **현재 최신.** 이모지 자리표시자 전부 제거 → **시드 기반 SVG 추상 아트 썸네일.** 우측 사이드바(최근 생성), 좌/우 사이드바 접기+메인 반응형, **FLIP 공유요소 라이트박스**(클릭 시 그 자리에서 풀스크린 확대).

### 3. v5 갤러리 버그 수정 (마지막 작업)
- **순환**: `o=i-gi` → 최단경로 래핑(-n/2~n/2)으로 무한 순환(끝→처음 자연스럽게).
- **클릭 신뢰성**: 드래그 끝 click이 재정렬하던 문제 → `dragged` 플래그로 드래그/클릭 구분.
- **"한 바퀴 회전"**: 의미불명한 트랙 360° 회전 → **전체 카드 자동 순회**("전체 둘러보기")로 변경.

### 4. 에러 2종 설명 (오너 질문)
- `CLAUDE_CODE_MAX_OUTPUT_TOKENS=4096`: **출력 길이 제한**. 큰 파일 한 번에 못 뱉음 → 파일을 조각내 Edit로 이어붙이는 방식으로 우회. (해결: `setx`로 상향 권장)
- `API Error: Overloaded`: **Anthropic 서버 일시 과부하(503)**. 코드 무관. 끊겨도 파일은 직전까지 저장되어 이어서 진행 가능.
